# ERP 模块编号说明

本文档用于固定 ERP 项目的模块编号。后续沟通需求、修改功能、拆分任务、写 Git 分支或提交说明时，可以直接使用编号。

例如：

```text
修改 1.1：人员管理增加岗位字段
修改 4.2：报价明细增加厚度加价规则
修改 3.1：项目订单必须选择客户项目
```

## 1. 基础资料

| 编号 | 模块 | 说明 |
| --- | --- | --- |
| 1.1 | 人员管理 | 维护员工、角色类型、工序、电话、状态、备注等基础人员资料 |
| 1.2 | 账号权限 | 维护登录账号、姓名、角色、启用状态、密码重置 |
| 1.3 | 产品资料 | 维护产品编码、类型、风格、名称、材质、颜色、价格、规格、图片 |
| 1.4 | 材料资料 | 维护材料编码、名称、类型、颜色、尺寸、状态、图片 |
| 1.5 | 工艺规则 | 维护生产工艺规则、工艺说明、状态 |
| 1.6 | 价格规则 | 维护报价加价规则、规则分类、计价模式、单价、上下限 |

## 2. 客户管理

| 编号 | 模块 | 说明 |
| --- | --- | --- |
| 2.1 | 客户档案 | 维护客户名称、电话、地址、等级、负责人、客户扩展信息 |
| 2.2 | 跟进记录 | 记录客户沟通、回访计划、跟进人、跟进时间 |
| 2.3 | 需求管理 | 记录客户需求类型、预算、风格偏好、详细需求 |
| 2.4 | 客户项目 | 按客户归集项目，维护项目名称、地址、状态、项目负责人 |

## 3. 订单管理

| 编号 | 模块 | 说明 |
| --- | --- | --- |
| 3.1 | 项目订单 | 新建和维护工厂订单、客户订单号、项目归属、客户快照信息 |
| 3.2 | 订单状态 | 查看和流转订单状态，记录状态日志 |
| 3.3 | 交期管理 | 维护计划交付时间、交期预警、延期原因 |
| 3.4 | 负责人分配 | 分配客服、深化设计、下单、排产、生产负责人 |

## 4. 报价管理

| 编号 | 模块 | 说明 |
| --- | --- | --- |
| 4.1 | 报价单主表 | 管理报价单、合同关联、总金额、报价状态、创建人 |
| 4.2 | 报价明细 | 维护产品、材料、尺寸、数量、面积、基础单价、加价规则、最终金额 |
| 4.3 | 版本管理 | 管理报价版本、历史版本、确认版本、变更记录 |
| 4.4 | 财务确认 | 审核收款、上传凭证、确认到账、推动订单进入排产 |

## 5. 排单管理

| 编号 | 模块 | 说明 |
| --- | --- | --- |
| 5.1 | 订单管理 | 生产前审核入口，承接已确认报价和已收款订单 |
| 5.2 | 板材清单 | 拆分板件尺寸、材质、颜色、厚度、数量、纹路方向 |
| 5.3 | 五金清单 | 拆分铰链、滑轨、拉手、连接件等五金明细 |
| 5.4 | BOM 清单 | 生成产品结构、材料用量、工艺用量、生产领料依据 |

## 6. 生产管理

| 编号 | 模块 | 说明 |
| --- | --- | --- |
| 6.1 | 生产任务 | 下发生产任务单，维护任务编号、合同、负责人、状态 |
| 6.2 | 排产计划 | 安排设备、人员、班次、生产顺序、计划时间 |
| 6.3 | 工序进度 | 跟踪开料、封边、打孔、组装、质检等工序进度 |
| 6.4 | 质检包装 | 记录质检结果、返工情况、包装信息、出库准备 |

## 7. 采购库存

| 编号 | 模块 | 说明 |
| --- | --- | --- |
| 7.1 | 采购申请 | 根据缺料或库存预警发起采购申请 |
| 7.2 | 采购订单 | 维护供应商、采购明细、到货状态、采购金额 |
| 7.3 | 库存管理 | 管理材料、半成品、成品库存数量和库位 |
| 7.4 | 出入库记录 | 记录采购入库、生产领料、退料、成品出库 |

## 8. 财务数据

| 编号 | 模块 | 说明 |
| --- | --- | --- |
| 8.1 | 收款管理 | 管理合同收款、收款计划、到账记录、核销状态 |
| 8.2 | 付款管理 | 管理采购付款、费用付款、付款审核 |
| 8.3 | 成本核算 | 汇总材料成本、人工成本、制造费用、订单成本 |
| 8.4 | 经营看板 | 查看订单额、回款率、毛利率、产能利用率等经营指标 |

## 使用约定

后续提出需求时，建议使用下面格式：

```text
修改 编号：具体需求
```

示例：

```text
修改 1.1：人员管理增加身份证号和入职日期
修改 4.2：报价明细增加非标颜色按百分比加价
修改 5.2：板材清单增加纹路方向字段
```

如果一个需求涉及多个模块，可以这样写：

```text
联动 3.1 + 4.1：项目订单创建后自动生成报价单草稿
联动 4.2 + 5.4：已确认报价明细生成 BOM 清单
```

## Git 分支命名建议

如果后续使用 Git 管理修改，分支可以按编号命名：

```text
feature/1.1-staff-management
feature/4.2-quote-detail
feature/5.2-board-list
feature/4.2-5.4-quote-to-bom
```

提交说明也可以带编号：

```text
1.1 人员管理增加岗位字段
4.2 报价明细增加厚度加价规则
5.2 板材清单增加纹路方向
```

## 当前优先试行建议

结合当前项目现状，建议优先试行和整理：

| 优先级 | 编号 | 模块 | 原因 |
| --- | --- | --- | --- |
| P0 | 4.2 | 报价明细 | 报价逻辑是后续拆单和生产的输入 |
| P0 | 4.1 | 报价单主表 | 需要明确报价状态、金额、版本入口 |
| P1 | 4.4 | 财务确认 | 收款确认后才能进入生产和排单 |
| P1 | 3.1 | 项目订单 | 报价必须绑定订单和项目 |
| P2 | 5.4 | BOM 清单 | 拆单后续核心输出 |
| P2 | 5.2 | 板材清单 | 全屋定制生产拆单重点 |
