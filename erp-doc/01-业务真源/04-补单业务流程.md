# 补单业务流程

> 文档性质：业务真源
>
> 状态：客户原因补单至客服下料放行已经确认
>
> 当前边界：流程结束于“客服已放行、等待下料”，不定义 BOM、锁库、实际下料和生产操作

## 1. 定义与适用范围

补单用于处理原客户订单成立后，由客户新增内容、客户修改要求或客户原因造成的重做。工厂责任造成的整单重做或局部重做属于售后，不进入补单流程。

补单不是新的客户订单，必须同时关联：

- 原客户订单。
- 原工厂订单。
- 本次补单 CAD、图片或其他客户资料。

补单金额计入原客户订单总应收和剩余应收。

## 2. 补单编号

补单生成独立工厂订单号，格式为：

```text
BD + 原生产线前缀 + yyyyMMdd + 当日三位序号
```

示例：

```text
BDL20260711001  实木生产线补单
BDV20260711001  板式生产线补单
```

- `BD` 表示补单。
- `L`、`V` 来自生产线基础资料，不在页面写死。
- 补单号全系统唯一，创建后不可修改；录入错误时作废并重新创建。
- 补单必须保存 `parent_factory_order_id`，指向原工厂订单。

## 3. 正常流程

```text
客服收到补单需求
  -> 选择原客户订单和原工厂订单
  -> 上传本次补单 CAD 或相关资料
  -> 补单进入 CAD 评审池
  -> 深化工程师领取并完成拆单
  -> 系统生成 BDL 或 BDV 工厂订单
  -> 补单进入对应报价流程
  -> 客户确认补单报价
  -> 财务处理补单应收或挂账确认
  -> 完成必要的厂长下料放行审批
  -> 客服执行补单放行
  -> 补单进入已放行、待下料
```

补单的 CAD 领取人与报价负责人不默认相同。补单报价负责人仍可由客服指定，或进入报价池由有权限人员领取。

## 4. 报价规则

- `BDL` 使用 ERP 内部完整报价引擎、报价版本、PDF和客户确认流程。
- `BDV` 使用板式外部报价系统，ERP 按工厂订单登记金额、附件、版本和客户确认。
- L 与 V 分别报价、分别确认，但补单应收统一进入原客户订单财务汇总。
- 补单报价发生修改时生成新版本，旧 PDF 和旧客户确认按报价版本规则失效。
- 客服可调整客户固定折扣或补单工厂订单的单次折扣，但必须保留完整变更记录。
- 客户订单最终成交价调整、议价和抹零仍由厂长审批。

## 5. 财务处理

### 5.1 原客户订单已经全额付清

- 补单金额形成新的待收应收。
- 客户实际付款后，由财务确认到账。
- 补单金额全部到账后，按全款规则处理，不再重复进行厂长例外下料审批。
- 客服取得放行权并执行补单放行。

### 5.2 原客户订单仍有欠款

- 补单金额继续计入原客户订单总应收和剩余应收。
- 补单本次可以不新增到账记录。
- 财务必须完成一次“同意挂账放行”确认，财务只确认挂账事实，不代替厂长审批。
- 厂长与老板沟通后，由厂长在系统记录沟通结果并完成下料放行审批。
- 审批通过后由客服执行补单放行。

### 5.3 月结客户

- 正常且未逾期的月结客户按月结规则处理。
- 客户已处于月结逾期锁定时，补单允许报价，但下料必须由厂长例外审批。
- 补单应收并入原客户订单和客户级月结欠款统计。

## 6. 放行范围与原订单关系

- 原客户订单已经放行，不代表后来新增的补单自动放行。
- 补单在原订单放行前生成时，应进入该次客户订单应收计算和未放行工厂订单快照。
- 补单在原订单放行后生成时，必须重新建立独立的放行周期。
- 新放行周期只包含当前仍未放行的有效工厂订单，不重复包含已经放行的原工厂订单。
- 同一放行周期内，不允许客服随意选择部分当前未放行订单；应对当次全部未放行有效订单统一处理。
- 已作废的补单不进入应收、审批或放行快照。

## 7. 必须留存的信息

- 原客户订单 ID。
- 原工厂订单号。
- 补单工厂订单号和生产线。
- 补单原因、客户说明和客服备注。
- 本次 CAD、图片及其他附件版本。
- CAD 领取、拆单、报价负责人和转交记录。
- 报价版本、折扣变更、PDF和客户确认记录。
- 补单金额、总应收变化和剩余应收变化。
- 财务到账确认或挂账确认。
- 厂长与老板沟通记录、审批结果和客服放行记录。

## 8. 权限边界

- 客服负责建立补单、关联原订单、上传资料、维护折扣、提交财务和执行放行。
- 深化工程师负责领取 CAD、拆单；成为当前报价负责人后才可编辑 L 报价。
- 财务负责确认实际到账或同意挂账，不决定补单技术范围，也不代替厂长审批。
- 厂长负责欠款补单和月结逾期补单的例外下料审批，并负责与老板沟通。
- 管理员可以处理系统异常，但不能代替正常业务角色完成确认和审批。

## 9. 售后边界

- 客户新增内容、客户修改或客户原因重做：补单。
- 工厂责任导致的免费整单重做或局部重做：售后。
- 深化工程师对售后只有技术建议权，无权单独决定责任和重做范围。
- 售后责任判定、审批、成本归属和免费重做细节暂不确认，不阻塞当前下料前主线。
