# ERP 业务总览

> 文档性质：跨模块业务真源
>
> 当前范围：客户建单至客服完成下料放行
>
> 状态：已确认部分正式使用，后续生产流程待继续确认

## 1. 核心数据层级

```text
客户
  -> 客户订单
       -> 客户原始CAD及版本
       -> 一张或多张工厂订单
            -> 报价、设计、下料、生产及售后数据
```

- 客户订单直接属于客户，不再设置项目层。
- 客户订单号全系统唯一，前端显示客户订单号，数据库使用内部 `customer_order.id` 关联。
- 工厂订单号本身就是工厂订单主键，另设工厂订单名称说明拆分内容。
- 一张客户订单可以拆成多张工厂订单。

## 2. 当前端到端主流程

```text
客服选择客户
  -> 填写唯一客户订单号并上传原始CAD
  -> 客户订单进入CAD评审池
  -> 深化工程师领取CAD
  -> 领取人建立拆单草稿并提交拆单
  -> 生成L实木或V板式工厂订单
  -> 工厂订单分别进入对应报价流程
  -> L线完成ERP明细报价
  -> V线登记并确认外部系统报价
  -> L线生成ERP报价PDF并记录客户确认
  -> V线保留外部报价文件并单独记录客户确认
  -> 客户订单统一汇总财务应收
  -> 客服建立付款方案和付款计划
  -> 财务确认实际到账
  -> 厂长完成必要审批
  -> 客服整张客户订单下料放行
  -> 客户订单进入待下料队列
```

## 3. 三个业务术语

- **拆单**：深化工程师查看客户原始 CAD，将一张客户订单拆成一张或多张工厂订单。
- **报价**：围绕具体工厂订单建立产品、尺寸、数量、特殊加价、五金和折扣明细。
- **下料**：收款和审批条件满足后，将客户订单正式推入生产准备，不属于拆单或报价。

## 4. 生产线边界

- L 表示实木生产线，使用 ERP 完整报价引擎。
- V 表示板式生产线，现阶段使用外部报价系统，ERP 按工厂订单登记结果。
- L 与 V 分别报价、分别对客确认，但在客户订单层统一形成总应收并进入财务。
- 生产线名称、编号前缀、报价模式、启停状态和排序必须通过基础资料维护，不写死在报价页面。

## 5. 补单与售后边界

- 客户新增内容或客户原因造成的重做，按补单处理。
- 补单使用 `BD` 加原生产线前缀的工厂订单号，并关联原工厂订单。
- 补单沿用建单、CAD评审、拆单和报价主线，金额并入原客户订单应收；原订单放行后新增的补单必须单独完成财务处理和放行。
- 工厂责任造成的整单重做或局部板件重做，按售后处理。
- 售后通常免费；深化工程师只有技术建议权，最终责任和重做范围仍待深化工程师、客服与厂长共同确认具体机制。

## 6. 当前流程终点

本阶段的终点是取得下料资格，但尚未执行实际下料：

- 所有需要的 L 报价已经完成。
- 所有 V 外部报价已经登记并确认。
- 折扣修改已经完成留痕，成交价调整已经完成厂长审批。
- 当前有效 L 报价 PDF 已经客户确认。
- 当前有效 V 外部报价已经分别确认。
- 系统形成唯一的客户订单总应收和当前付款计划。
- 财务已经按实际情况确认到账，或订单符合月结、免单等放行条件。
- 必要的付款计划审批和下料放行审批已经完成。
- 客服已经执行整张客户订单下料放行。
- 客户订单进入“已放行、待下料”。

BOM、锁库、采购、实际下料、排产和后续生产流程继续在下一批业务文档中确认。
