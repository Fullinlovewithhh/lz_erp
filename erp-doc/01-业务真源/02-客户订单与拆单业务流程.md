# 客户订单与拆单业务流程

> 文档性质：业务真源
>
> 状态：客户建单、CAD评审和拆单部分已经确认

## 1. 文档范围

本文档固定以下业务流程：

```text
客户
  -> 新建客户订单并上传 CAD
    -> CAD 评审池
      -> 深化工程师领取
        -> 拆单
          -> 生成工厂订单
            -> 进入报价池
```

本文档不包含工厂订单内部报价、客户报价确认、财务收款、下料、生产和售后流程。上述内容分别在后续业务文档中确认。

## 2. 正式数据层级

当前正式订单层级为：

```text
客户 customer
  -> 客户订单 customer_order
    -> 工厂订单 factory_order（一张或多张）
```

客户下面直接建立客户订单，客户订单下面建立一张或多张工厂订单。

## 3. 客户规则

- 后端使用 `customer.id` 作为客户唯一身份。
- 前端只显示客户名称，不显示内部数字 ID。
- 客服通过客户名称搜索和选择客户。
- 一个客户可以创建多张客户订单。
- 客户已有业务记录后不得物理删除，只能停用。

## 4. 新建客户订单

### 4.1 操作流程

新建客户订单与上传客户原始 CAD 合并为一次操作：

```text
客服打开“新建客户订单”
  -> 选择客户姓名
    -> 填写客户订单号
      -> 上传客户原始 CAD 及辅助资料
        -> 点击创建
          -> 后端生成 customer_order.id
            -> 保存订单和全部附件
              -> 客户订单直接进入 CAD 评审池
```

### 4.2 必填内容

- 客户姓名
- 客户订单号
- 至少一个 CAD 文件

不提供“无 CAD 暂存草稿”功能。以上必填内容不完整时不能创建客户订单。

### 4.3 原子性要求

客户订单和首批附件必须在同一次业务操作中成功：

- 订单保存成功但附件保存失败时，整次创建应当回滚。
- 附件上传成功但订单保存失败时，不得留下无归属的正式附件。
- 创建成功后自动记录跟单客服和创建时间。
- 创建成功后评审状态设置为“待领取”。

## 5. 客户订单号

- 客户订单号由客户提供，可以由汉字、字母和数字组成。
- 后端自动生成的 `customer_order.id` 才是客户订单的唯一主键。
- 前端不显示内部数字 ID，只显示客户订单号。
- 客户订单号不作为数据库表之间的关联键。
- 客户订单号全系统不允许重复。
- 前端发现重复订单号时直接阻止创建，后端再次校验，数据库使用唯一约束兜底。
- 查询结果同时显示客户名称和客户订单号。
- 客户订单号前后空格应自动清理。

数据库使用 `UNIQUE(customer_order_no)` 保证并发创建时也不会产生重复订单号。

## 6. 客户原始资料

### 6.1 附件范围

新建客户订单时允许一次上传多个文件：

- CAD：`dwg`、`dxf`
- 文档：`pdf`
- 图片：`jpg`、`jpeg`、`png`
- 压缩包：`zip`、`rar`、`7z`

首批附件中至少存在一个 CAD 文件。PDF、图片和压缩包只能作为辅助资料，不能替代必需的 CAD 文件。

### 6.2 归属关系

- 客户传来的原始 CAD 和辅助资料绑定 `customer_order.id`。
- 文件名不能用于建立订单关联关系。
- 后续深化图和生产图属于具体工厂订单，不属于本阶段客户原始资料。

## 7. CAD 版本

- 客户后续发送修改版资料时，不覆盖、不删除原文件。
- 每次追加形成新的客户资料版本，例如 V1、V2、V3。
- 每个版本记录上传人、上传时间、文件列表和版本备注。
- 最新提交版本标记为当前有效版本，旧版本只读保留。
- 新版本上传后通知当前 CAD 评审设计师和跟单客服。
- 已完成的评审、拆单意见和拆单草稿标记为“资料已更新，待重新确认”。
- 已生成的正式工厂订单不自动删除或失效。

### 7.1 已拆单后的处理方式

CAD 更新时，如果已经生成正式工厂订单，由客服选择以下处理方式：

1. 不影响原拆单：补充资料或小修改，继续使用现有工厂订单。
2. 重新拆单：原工厂订单标记为已取消或被替代，重新评审并生成新工厂订单；历史记录不删除。
3. 作为补单处理：原工厂订单继续有效，新增内容进入补单流程。

系统必须记录处理方式、操作人、处理时间和备注。

## 8. CAD 评审池

### 8.1 领取规则

```text
客户订单创建成功
  -> 进入 CAD 评审池
    -> 深化工程师领取
      -> 谁领取，谁负责查看 CAD 并完成拆单
```

CAD 评审只保留必要记录：

- 领取人
- 领取时间
- 拆单意见
- 拆单草稿提交时间

不建设复杂的退回申请、指定转交和多级审批流程。

### 8.2 人员边界

- 领取 CAD 的深化工程师负责判断应拆成多少张工厂订单。
- 领取 CAD 的深化工程师负责建立拆单草稿。
- CAD 评审设计师不默认成为后续报价负责人。
- 客服可以查看 CAD 领取人和拆单进度，但不需要确认拆单。

## 9. 业务术语

系统统一使用以下口径：

### 9.1 拆单

查看客户 CAD，并将一张客户订单拆成一张或多张工厂订单，叫“拆单”。

### 9.2 报价

正式工厂订单生成后，在工厂订单内部拆分产品、尺寸、材料、工艺和五金并计算价格，叫“报价”。

### 9.3 下料

客户确认报价并完成对应付款、财务确认和必要审批后，客服将订单推向生产，之后生成 BOM、板件和生产任务，叫“下料”。

拆单、报价和下料是三个不同阶段，页面名称、状态名称、接口名称和文档不得混用。

## 10. 拆单草稿

每张工厂订单草稿至少包含：

- 工厂订单名称，例如“住宅1-客厅”
- 生产线
- 拆单说明，可选
- 排列顺序

一张客户订单可以拆成一张或多张工厂订单：

```text
客户订单：住宅1
  -> 住宅1-客厅 / 实木生产线
  -> 住宅1-卧室 / 板式生产线
  -> 住宅1-厨房 / 板式生产线
```

- 深化工程师可以反复保存拆单草稿。
- 正常情况下，确认无误后一次提交全部拆单草稿。
- 确有需要时允许后续执行“追加拆单”。
- 追加拆单必须记录操作人、时间和追加原因。
- 追加拆单生成正常工厂订单，不自动视为补单。

## 11. 生产线与工厂订单号

前端显示生产线中文名称：

| 前端名称 | 内部编号前缀 | 编号示例 |
|---|---|---|
| 实木生产线 | `L` | `L20260713001` |
| 板式生产线 | `V` | `V20260713001` |

生产线属于可维护基础资料，后续允许新增、停用或调整排序。每条生产线至少维护：

- 生产线名称
- 工厂订单号前缀
- 是否启用
- 排序

工厂订单号由系统根据生产线前缀、日期和当日序号自动生成，用户不手工填写。

## 12. 提交拆单

深化工程师提交拆单后：

1. 系统校验每条草稿的工厂订单名称和生产线。
2. 系统一次性生成正式工厂订单号。
3. 正式工厂订单绑定原客户订单。
4. 工厂订单直接进入报价池。
5. 客户订单状态更新为“已拆单”或“报价中”。
6. 客服收到拆单完成提醒，并可以查看全部工厂订单和报价进度。
7. 拆单深化工程师不默认成为报价负责人。

拆单不需要客服确认。

## 13. 拆单错误与报价撤回

### 13.1 尚未领取报价

- 深化工程师可以修改工厂订单名称和拆单说明。
- 修改名称不改变工厂订单号。
- 修改生产线时，原工厂订单号作废，按新生产线生成新工厂订单号。
- 作废编号保留历史且不得重新使用。

### 13.2 已领取但尚未提交报价

```text
拆单深化工程师发起撤回
  -> 填写撤回原因
    -> 当前报价停止编辑
      -> 工厂订单退回“拆单待修正”
        -> 修正后重新进入报价池
```

- 报价草稿不删除。
- 只修改名称时，原报价草稿可以继续使用。
- 修改生产线并生成新编号时，可以复制原草稿，但报价负责人必须重新核对。
- 领取、撤回和重新进入报价池的历史永久保留。

### 13.3 已提交报价但客户尚未确认

- 当前报价版本标记为“已撤回”。
- 客户订单报价汇总重新计算。
- 已生成的报价 PDF 标记为“已失效”。
- 工厂订单退回拆单修正，修正后重新报价。
- 原报价版本不得覆盖或删除。
- 该阶段由客服执行撤回，因为已经涉及客户报价汇总和对外沟通。

### 13.4 客户已经确认或已经付款

禁止使用普通报价撤回，必须进入正式订单变更流程：

```text
发起订单变更
  -> 客服确认客户意见
    -> 必要时生成新报价版本
      -> 原客户确认和 PDF 失效
        -> 重新确认应收和剩余款项
```

客户新增内容进入补单流程；工厂自身错误根据后续售后规则处理。

## 14. 客户是否成交

正常业务不在 CAD 评审或拆单阶段取消。客户通常在完整报价出来后决定是否制作：

```text
客户接受报价
  -> 进入客户确认、付款和下料流程

客户暂未决定
  -> 状态为“待客户确认”

客户明确不做
  -> 状态为“报价未成交”
```

客户不做时：

- 客服填写一条自由备注，不设置固定未成交原因选项。
- 客户订单、CAD、拆单记录、工厂订单、报价版本和 PDF 全部保留。
- 订单不得进入付款、下料和生产流程。
- 后续客户重新考虑时，可以基于历史记录生成新报价版本。

只有明显录错、重复创建等非业务数据，才允许管理员在尚未产生工厂订单前软删除。

## 15. 本流程结束状态

本流程完成后，应形成以下正式数据：

- 一张客户订单
- 一组有版本记录的客户原始资料
- 一名 CAD 评审深化工程师
- 一组拆单草稿和提交记录
- 一张或多张正式工厂订单
- 每张工厂订单对应的生产线和工厂订单号
- 工厂订单进入报价池的状态记录

下一阶段由《工厂订单报价业务流程》继续定义。
