# 工厂订单报价业务流程

> 文档性质：业务真源
>
> 状态：工厂订单报价至客户确认部分已经确认

## 1. 文档范围

本文档固定工厂订单从进入报价阶段，到客户确认并移交财务的完整业务规则。

```text
工厂订单生成
  -> 根据生产线确定报价方式
    -> 完成报价
      -> 生成客户报价文件
        -> 客户确认
          -> 客户订单进入付款计划
```

本文档不定义收款、下料、库存锁定和生产流程。

## 2. 生产线报价边界

生产线必须配置报价模式，不能在代码中根据生产线名称写死。

```text
ERP       ERP内部报价
EXTERNAL  外部系统报价
NONE      不需要报价
```

当前生产线配置：

| 编号前缀 | 生产线 | 报价模式 |
|---|---|---|
| `L` | 实木生产线 | `ERP` |
| `V` | 板式生产线 | `EXTERNAL` |

### 2.1 L实木生产线

- L工厂订单进入ERP报价订单池。
- 使用ERP产品、特殊计价规则、五金、折扣和版本体系。
- ERP生成完整客户报价PDF并记录客户确认。

### 2.2 V板式生产线

- V工厂订单不进入ERP报价订单池。
- V报价继续在板式生产线现有系统中完成。
- ERP不重复维护V产品报价明细，也不重新计算V金额。
- 每张V工厂订单由客服人工登记外部报价结果。
- V报价与L报价分开对外确认，但财务统一在ERP管理。
- ERP后续承接V订单的仓库、库存、物料和下料放行数据。

每张V工厂订单登记：

- 外部报价单号
- 外部报价最终金额
- 外部报价日期
- 外部报价单附件
- 客户确认状态和时间
- 登记客服和登记时间
- 修改历史

## 3. 报价单位和版本

- 每张L工厂订单分别报价。
- 一张L工厂订单只有一个当前有效报价版本。
- 一张报价中可以包含多条产品明细。
- 每条产品明细可以包含多条特殊加价。
- 草稿反复保存时继续使用当前草稿版本。
- 正式提交后再次修改必须生成新版本。
- 历史版本只读，不得覆盖或物理删除。
- 报价负责人转交后，草稿、明细和附件继续跟随工厂订单保留。

## 4. 产品类别和产品资料

产品类别和产品不能写死在报价页面或Java代码中。

```text
产品类别资料
  -> 产品资料
    -> 报价明细
```

- 新增产品类别后，报价页面自动出现。
- 停用类别后，新报价不能选择，历史报价继续显示原名称。
- 每个具体产品只允许一种计价方式。
- 同一业务产品需要另一种计价方式时，应建立另一个具体产品。

产品资料至少保存：

- 产品类别
- 产品名称
- 唯一计价方式
- 计价单位
- 基础单价
- 报价时需要填写的尺寸字段
- 最低计价面积或长度，可选
- 默认是否参与折扣
- 是否启用

支持的计价单位至少包括：

```text
平方米、延米、米、个、套、樘、块、其他可维护单位
```

## 5. 报价明细粒度

采用“相同规格合并，不同规格分行”的规则：

- 产品、尺寸、材料、颜色、工艺、基础单价全部相同，可以合并数量。
- 任意一项不同，必须建立另一条报价明细。
- 报价阶段不拆到每一块物理板件。
- 逐板件编号和二维码留到付款后的下料与生产阶段。

例如：

```text
实木门板A：600×800×20，数量6块
实木门板B：500×700×20，数量4块
```

## 6. 自动计价

产品资料决定报价页面显示哪些输入项以及如何计算。

### 6.1 按面积

```text
填写宽度、高度、数量
实际面积 = 宽度 × 高度 × 数量
系统按最低计价规则得到计价面积
基础金额 = 计价面积 × 基础单价
```

### 6.2 按长度

```text
填写长度、数量
实际长度 = 长度 × 数量
系统按最低计价规则得到计价长度
基础金额 = 计价长度 × 基础单价
```

### 6.3 按数量

```text
填写数量
基础金额 = 数量 × 基础单价
```

- 报价人员填写尺寸和数量，不直接填写产品最终金额。
- 普通报价人员不能修改产品基础单价。
- 产品价格错误时修改产品资料，只影响以后新建的报价明细。
- 历史报价保存产品类别、名称、单位、单价和计算结果快照。

## 7. 特殊计价规则

特殊加价表示在产品基础价格之上增加的费用，不是重新填写完整产品价格。

例如：

```text
实木门板基础单价：500元/㎡（包含正常油漆）
描金附加工艺：增加80元/㎡
最终计价单价：580元/㎡
```

### 7.1 常用特殊计价规则

在基础资料中维护“特殊计价规则”，内容不写死在代码中。

每条规则保存：

- 规则名称
- 规则分类
- 计价方式
- 加价金额
- 计价单位
- 默认是否参与折扣
- 适用产品类别，可选
- 备注
- 是否启用
- 创建人和修改时间

计价方式至少支持：

```text
每平方米加价
每延米加价
每个/套/樘/块加价
固定一次性加价
```

常用分类可在基础资料中自行维护，初始可以包括：

- 油漆附加工艺
- 特殊尺寸
- 特殊造型
- 特殊加工
- 特殊材料
- 专用刀具与模具
- 其他加价

分类可以新增、改名、排序和停用。历史报价保存分类和规则快照。

### 7.2 一次性特殊加价

不常用的特殊情况直接在报价明细中填写，不加入基础资料规则库。

必须填写：

- 加价分类
- 加价名称
- 加价原因
- 计价方式
- 加价值或固定金额
- 是否参与折扣

系统自动记录填写人、时间、当前报价版本和可选附件。

例如：

```text
分类：专用刀具与模具
名称：本单专用雕刻刀
原因：客户特殊造型需要临时购买
计价方式：固定一次性加价
金额：800元
参与折扣：否
```

### 7.3 绑定关系

- 所有特殊加价统一绑定具体报价明细。
- 不单独建立工厂订单级特殊加价。
- 服务整张工厂订单的一次性费用，绑定第一条相关产品明细并只计算一次。
- 报价主表自动汇总并展示特殊加价名称、金额和合计。
- 特殊加价自动属于当前正在编辑的报价版本，前端不提供版本选择控件。

## 8. 五金报价

- 五金从五金资料中选择。
- 系统自动带出名称、规格、图片、单位和报价单价。
- 报价人员只填写数量。
- 系统计算五金金额。
- 五金加入工厂订单报价总额，但不参与工厂订单折扣。
- 报价人员不能临时修改五金单价。
- 五金价格错误时统一修改五金资料，历史报价保存价格快照。
- 报价阶段不锁定库存。
- 客户确认、财务和下料放行后，才在下料阶段锁定五金库存。
- 库存不足时由下料阶段生成采购需求。

## 9. 折扣

折扣只作用于：

- 标记为参与折扣的产品
- 标记为参与折扣的特殊加价

折扣不作用于：

- 五金
- 标记为不参与折扣的产品
- 标记为不参与折扣的特殊加价

### 9.1 默认规则

- 油漆、尺寸、造型和加工类特殊加价默认参与折扣。
- 特殊材料、刀具和模具类默认不参与折扣。
- 报价人员可以逐条调整特殊加价是否参与折扣。

### 9.2 固定客户折扣

客户档案维护当前默认折扣、有效期、修改人、修改时间和备注。

- 固定折扣默认应用到该客户所有L工厂订单。
- 新客户默认十折。
- 客服可以直接新增或修改客户固定折扣，不经过销售经理或老板审批。
- 每次修改必须记录修改前折扣、修改后折扣、原因、客服和时间。
- 折扣修改只影响修改后生成的新报价版本，不覆盖历史报价。

### 9.3 单张工厂订单折扣

```text
客服选择一张L工厂订单
  -> 修改该订单折扣并填写原因
    -> 系统记录修改前后折扣、客服和时间
      -> 生成新报价版本
```

- 单张工厂订单折扣只影响该订单，不修改客户固定折扣。
- 单张订单折扣由客服直接修改，不设置销售经理或老板审批节点。
- 客服不能通过折扣入口修改产品基础单价、尺寸、数量或技术明细。
- 折扣修改后，旧PDF和旧客户确认失效。

## 10. 报价计算顺序

```text
产品基础金额
+ 参与折扣的特殊加价
= 参与折扣原价

参与折扣原价 × 工厂订单折扣率
+ 不参与折扣的产品
+ 不参与折扣的特殊加价
+ 五金金额
= 工厂订单最终金额
```

L客户报价汇总：

```text
全部L工厂订单最终金额
+ L报价成交价调整
= L未税报价合计

L未税报价合计 × 税率
= 税额

L未税报价合计 + 税额
= L价税合计
```

税率规则：

- 基础报价默认不计税，报价总表的税率默认为 `0%`。
- 税率不是固定值，报价总表允许按本次客户报价填写实际百分比。
- 填写 `7` 表示 `7%`，填写 `8` 表示 `8%`，系统换算后计算税额。
- 税率在生成最终客户报价 PDF 之前确定，并显示在 PDF 中。
- 不计税时，客户最终应付为未税报价合计；计税时，客户最终应付为价税合计。
- 付款计划以客户确认的当前 PDF 最终应付金额为准。
- PDF 生成后修改税率，必须生成新的 PDF 版本；旧 PDF 和旧客户确认按版本规则失效。

成交价调整可以用于最终议价和抹零，但必须生成调整版本并经过厂长审批。已经确认的到账记录不因后续报价修改而删除。

## 11. L客户报价PDF

同一客户订单下的所有L工厂订单必须一起生成PDF并整单确认，不允许只确认部分L工厂订单。

PDF必须显示全部客户价格细节。

### 11.1 汇总信息

- 公司名称、Logo、地址和电话
- 客户名称和客户订单号
- 报价日期、有效期和报价版本
- 客服和报价负责人
- 是否含税、税率、税额和价税合计
- 每张L工厂订单号、名称和最终金额
- L实木报价总额

### 11.2 工厂订单和产品明细

- 工厂订单原价、折扣率、折扣金额、不参与折扣金额、五金金额和最终金额
- 产品类别、产品名称和图片
- 材料、颜色和规格
- 宽、高、厚、数量和单位
- 实际面积或长度、计价面积或长度
- 基础单价和基础金额
- 每条特殊加价分类、名称、计价方式、计价数量、金额和折扣资格
- 五金名称、规格、图片、单位、数量、单价和金额
- “五金不参与折扣”的明确说明
- 明细备注

V板式报价不出现在L实木PDF中，由板式系统单独提供报价文件。

## 12. 客户确认

客户对全部L工厂订单整单确认。

必填记录：

- 当前有效PDF版本
- 确认方式
- 确认时间
- 客户联系人
- 记录客服

选填记录：

- 确认备注
- 微信截图
- 邮件、签字文件或其他附件

规则：

- 只有全部L工厂订单完成正式报价，才能生成L汇总PDF。
- 客服只能确认当前有效PDF。
- 任意L工厂订单报价发生修改，旧PDF和旧客户确认自动失效。
- 新版本必须重新生成完整PDF并让客户整单确认。
- 客户明确不做时，客户订单标记为“报价未成交”，客服填写自由备注。
- 所有失效版本和确认记录永久保留。

## 13. 进入财务的条件

客户订单总应收：

```text
已确认的L实木报价金额
+ 已确认的全部V板式外部报价金额
= 客户订单总应收
```

只有以下条件全部满足，客户订单才标记为“报价已完成”并进入正式付款计划：

- 所有L工厂订单报价完成。
- L实木汇总PDF已经客户整单确认。
- 所有V工厂订单已经登记外部报价金额。
- 所有V外部报价已经标记客户确认。
- 所有折扣修改已经完成留痕，成交价调整已经完成厂长审批。
- 不存在正在修改或已失效但尚未重新确认的报价。

客户订单只有L或只有V时，另一种报价自动视为不适用。

报价进入财务后仍允许修改，但必须重新生成报价版本、完成折扣留痕或必要的成交价审批、重新生成PDF并取得客户确认。已经确认到账的记录保持不变，系统重新计算总应收和剩余应收。

## 14. 权限边界

- 当前L报价负责人编辑产品、尺寸、数量、特殊加价和五金明细。
- 报价负责人可以是深化工程师或有报价权限的客服。
- 非当前负责人只能查看，管理员除外。
- 客服负责客户沟通、客户固定折扣和单张工厂订单折扣修改、PDF生成和确认记录。
- 客服修改折扣必须留痕，但不需要销售经理或老板审批。
- 厂长负责客户订单成交价调整审批。
- 财务读取已确认应收并管理后续付款，不修改报价明细。
