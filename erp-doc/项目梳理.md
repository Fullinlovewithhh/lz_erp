# ERP 项目梳理

本文用于重新梳理当前文件夹中的 Lonzvine 全屋定制工厂 ERP 项目，方便后续做代码清理、模块拆分、二次开发或交付部署。

## 1. 项目位置

当前工作区根目录：

```text
fac_erp_v1.0/
├─ gcerp kaifa/              # 实际 ERP 项目目录
├─ scripts/                  # 外层辅助脚本
├─ LV_ERP_1.2.zip            # 历史打包文件
└─ 产品资料_导出.csv          # 外层产品导出数据
```

实际项目主体在：

```text
gcerp kaifa/
```

## 2. 技术栈

前端：

- Vue 3
- Vite
- Axios
- 单页应用，目前主要逻辑集中在 `erp-web/src/App.vue`

后端：

- Spring Boot 3.3.5
- Java 21
- MyBatis-Plus 3.5.7
- MySQL

数据库：

- 数据库名：`gc_erp`
- 完整导出文件：`gc_erp_full.sql`
- 后端初始化文件：`erp-server/init.sql`

## 7. 前端现状

当前前端主要特点：

- `App.vue` 非常大，承担了登录、布局、菜单、表格、弹窗、报价、订单、财务等大量职责。
- `src/api/contract.js` 虽然文件名叫 `contract.js`，但实际封装了认证、客户、项目、基础资料、报价、付款、生产等几乎全部接口。
- 页面路由没有拆分，主要通过 `activeModule` 和 `activeSubModule` 做条件渲染。
- 已经抽出的组件较少：
  - `CustomerProjectProfile.vue`
  - `QuoteExtraPricePanel.vue`
  - `SpecialCraftDropdown.vue`

建议后续重构时先拆前端，因为这是当前维护成本最高的部分。

## 8. 后端现状

后端目录按业务模块拆分较清楚，基本结构是：

```text
Controller -> Service -> Mapper -> Entity/Request
```

优点：

- 模块边界比前端清楚。
- 接口路径统一在 `/api` 下。
- 已有统一返回 `ApiResponse` 和异常处理。
- 已有登录鉴权拦截和操作日志基础。

需要继续梳理的点：

- 数据库兼容变更目前在代码初始化逻辑中处理，后续建议迁移到正式 migration 工具或 SQL 版本脚本。
- 上传文件在 `erp-server/upload` 下，交付、备份和部署时要明确是否保留。
- `application.yml` 中包含本机数据库密码，后续建议改成环境变量或 `.env`。

## 9. 建议清理范围

重新整理项目时，可以优先清理这些生成物或本地文件：

```text
erp-web/node_modules/
erp-web/dist/
erp-server/target/
erp-web/*.log
erp-web/vite.config.js.timestamp-*.mjs
```

这些不属于源码核心，重新安装依赖或重新构建即可生成。

需要谨慎保留：

```text
gc_erp_full.sql
erp-server/upload/
erp-web/public/import-templates/
产品资料_导出.csv
LV_ERP_1.2.zip
```

其中 `upload` 里可能有历史图片，是否删除要看是否需要保留真实业务附件。

## 10. 建议重构顺序

### 第一阶段：项目清洁和可运行

目标：保证项目能干净启动、交付、恢复。

建议动作：

1. 校正 README 编码乱码。
2. 明确 JDK 版本，当前 `pom.xml` 使用 Java 21，但旧 README 写的是 JDK 17 建议。
3. 清理生成物：`node_modules`、`dist`、`target`、日志文件。
4. 增加 `.env.example` 或配置说明，避免数据库密码写死。
5. 明确数据库初始化使用 `gc_erp_full.sql` 还是 `erp-server/init.sql`。

### 第二阶段：前端模块拆分

目标：降低 `App.vue` 维护成本。

建议拆分：

```text
src/
├─ api/
│  ├─ auth.js
│  ├─ master.js
│  ├─ customer.js
│  ├─ contract.js
│  ├─ quote.js
│  ├─ payment.js
│  └─ production.js
├─ views/
│  ├─ LoginView.vue
│  ├─ master/
│  ├─ customer/
│  ├─ order/
│  ├─ quote/
│  ├─ finance/
│  └─ production/
├─ components/
│  ├─ layout/
│  ├─ table/
│  ├─ upload/
│  └─ quote/
└─ composables/
   ├─ useAuth.js
   ├─ useMasterData.js
   ├─ useOrders.js
   └─ useQuote.js
```

### 第三阶段：业务链路补齐

目标：从“客户到生产”形成稳定闭环。

建议优先级：

1. 客户档案和客户订单稳定化。
2. 合同订单字段、状态机、负责人分配补齐。
3. 报价版本管理补齐。
4. 财务确认和收款凭证规范化。
5. 生产任务状态和工序进度补齐。
6. 再进入 BOM、板材、五金、采购库存。

### 第四阶段：权限和审计

目标：让系统适合真实多人使用。

建议补齐：

1. 角色和菜单权限细化。
2. 操作日志覆盖新增、修改、删除、审核、状态流转。
3. 关键动作增加确认和撤销/作废机制。
4. 数据导入增加错误报告和重复数据策略。
