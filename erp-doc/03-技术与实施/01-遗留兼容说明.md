# 遗留兼容说明

> 文档性质：技术实施说明，不作为业务规则来源
>
> 整理日期：2026-07-18

## 1. 当前正式主线

新业务统一使用以下层级：

```text
customer.id
  -> customer_order.id
       -> factory_order.factory_order_id
```

- 客户订单不再关联项目层。
- 客户原始 CAD 绑定客户订单。
- 报价、报价任务和后续生产数据应绑定工厂订单号。
- L 使用 ERP 报价，V 使用外部报价登记，两者在客户订单层统一汇总。

## 2. 仍存在的兼容结构

- 旧 `contract` 相关实体、接口和前端代码仍有兼容读取用途。
- 旧写入入口已经停用，不应再产生新的正式业务数据。
- 前端旧“项目订单”页面代码仍可能保留在综合组件中，但当前菜单不再进入该页面。
- 旧项目相关后端结构只有在完成全量引用扫描、数据迁移和回归测试后才能物理删除。
- 当前实现若将 V 外部报价金额写入 L 客户报价 PDF，应按“报价分开、财务统一”的最新业务规则校正；V 金额只进入统一财务应收。

## 3. 数据库升级方式

- 全新数据库以 `erp-server/init.sql` 为基准。
- 已有数据库通过 `database/migrations` 中的版本 SQL 升级。
- Java 启动兼容初始化器只用于补齐兼容字段、索引和必要表结构，不能代替长期迁移版本管理。
- 每次正式部署前必须备份数据库，并在测试库验证迁移脚本。

## 4. 允许删除旧结构的条件

必须同时满足以下条件：

1. 代码中不再写入旧表或旧字段。
2. 所有历史数据已迁移并完成数量、金额和关联完整性核对。
3. 前端不再调用旧接口。
4. 后端新接口通过业务回归测试。
5. 数据库已经完成可恢复备份。
6. 至少经过一个稳定运行周期。

未满足以上条件时，只能隐藏旧入口、停用旧写入并保留兼容读取，不直接删表。
