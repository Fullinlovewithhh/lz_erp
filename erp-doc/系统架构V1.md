# 全屋定制 ERP 系统架构（V1）

## 1. 总体架构
- 前端：Vue3 + Element Plus（B/S）
- 后端：Spring Boot + MyBatis-Plus
- 数据库：MySQL
- 目标：围绕“客户-订单-报价-收款-排产-生产-采购库存-财务”形成闭环。

## 2. 功能架构（按你的目录）

### 2.1 基础资料
- 人员权限：用户、角色、菜单、数据权限
- 产品资料：柜体/门板/配件产品主数据
- 材料资料：板材、五金、封边、辅料
- 工艺规则：开料、封边、打孔、组装规则
- 价格规则：材料价、工艺价、系数、折扣规则

### 2.2 客户管理
- 客户档案：客户主数据、联系方式、地址
- 跟进记录：沟通记录、回访计划
- 需求管理：户型需求、风格偏好、预算范围

### 2.3 订单管理
- 合同订单：合同主表、合同附件
- 订单状态：待设计/待报价/待收款/已收款/待生产/生产中/已完成
- 交期管理：计划交付日、预警规则
- 负责人分配：客服、设计、下单员、车间负责人

### 2.4 报价管理
- 报价单主表：报价编号、合同关联、总金额
- 报价明细：按产品/材料/工艺明细报价
- 版本管理：V1/V2/V3 版本留痕
- 财务确认：确认到账后才允许排产

### 2.5 排单管理
- 订单管理：生产前审核入口
- 板材清单：板件尺寸、纹路方向、优化下料参数
- 五金清单：铰链、滑轨、拉手等
- BOM 清单：产品结构树、材料用量

### 2.6 生产管理
- 生产任务：任务单、工序单
- 排产计划：设备、人员、班次、顺序
- 工序进度：开料/封边/打孔/组装/质检
- 质检包装：质检结论、返工、包装出库

### 2.7 采购库存
- 采购申请：缺料触发请购
- 采购订单：供应商下单、到货跟踪
- 库存管理：原材料/半成品/成品库存
- 出入库记录：入库、领料、退料、出库

### 2.8 财务数据
- 收款管理：合同收款计划、到账核销
- 付款管理：采购付款、费用付款
- 成本核算：材料成本、人工成本、制造费用
- 经营看板：订单额、回款率、毛利率、产能利用率

## 3. 核心数据模型（主表建议）
- 基础资料：`sys_user` `sys_role` `sys_user_role` `sys_menu` `product` `material` `process_rule` `price_rule`
- 客户管理：`customer` `customer_followup` `customer_requirement`
- 订单管理：`contract` `contract_assignment` `contract_delivery`
- 报价管理：`quotation` `quotation_item` `quotation_version`
- 排单管理：`bom_header` `bom_item` `board_item` `hardware_item` `cut_plan`
- 生产管理：`production_task` `production_schedule` `process_report` `qa_pack_record`
- 采购库存：`purchase_request` `purchase_order` `purchase_order_item` `inventory` `stock_txn`
- 财务数据：`payment_record` `receipt_record` `cost_ledger` `finance_dashboard_cache`

## 4. 跨模块关键约束
- 财务确认未到账：禁止进入排产。
- 合同状态必须按状态机推进，禁止跳步。
- 报价多版本时，仅“最新已确认版本”可用于生产。
- BOM 冻结后方可下发生产任务，变更必须走变更单。

## 5. 接口分层建议
- 控制层（Controller）：参数校验、统一返回
- 服务层（Service）：业务规则、状态流转、事务
- 持久层（Mapper）：数据库访问
- 审计层：流程日志、操作日志、异常日志

## 6. 开发实施阶段（建议）

### 阶段A（当前优先）
- 完成：基础资料（人员权限简版）、客户管理、订单管理、报价管理、财务收款确认
- 目标：打通“客户 -> 合同 -> 报价 -> 收款确认”

### 阶段B
- 完成：排单管理 + 生产管理
- 目标：打通“已收款 -> 排产 -> 生产 -> 完工”

### 阶段C
- 完成：采购库存 + 成本核算 + 经营看板
- 目标：打通“物料供给 + 成本分析 + 经营分析”
